支持未来网络创新的可编程虚拟化路由器技术

2017.05.04 Wasdns  热度 ℃

支持未来网络创新的可编程虚拟化路由器技术

支持未来网络创新的可编程虚拟化路由器技术 - PEARL

PEARL通过灵活可编程性实现网络基础设施与功能的分离，通过虚拟化隔离机制实现网络基础设施拥有权与功能拥有权的分离，实现在单一物理网络平台上构建多个异构虚拟网络系统。

介绍

背景：

未来互联网体系需要在试验网络系统中进行评估与验证，但由于路由器的封闭性与正常业务对试验的影响，在现有的互联网上部署验证非常困难。

由此提出了网络虚拟化的思想：在一个实际的物理网络上构建多个独立且互不干扰的虚拟网络平面，以提供真实有效的并行实验平台。

为了支持网络虚拟化技术要求，路由器需满足：

  • 虚拟化与隔离性

  • 灵活的可扩展可编程机制

  • 高效数据包处理能力

本文贡献：

  • 讨论了当前可编程虚拟化路由器结构，分析了虚拟化、可编程以及高效数据包处理等关键技术；

  • 一种支持未来网络创新的可编程虚拟化路由器平台 - PEARL

可编程虚拟化路由器结构

传统路由器分为三层：

  • 硬件设备层

  • 操作系统层

  • 路由管理层

可编程路由器在操作系统层引入虚拟化技术，从而在同一硬件设备层上构建多个虚拟的路由管理层，从而实现多个并行的虚拟路由器。但是面临以下挑战：

  • 性能

  • 高效数据包处理

  • 灵活可扩展可编程

  • 隔离

核心问题 - 如何折衷考虑高效数据包处理与灵活可扩展可编程？

问题：高性能和高速度 <=> 灵活性和可扩展性

虚拟路由器采用数据平面与控制平面相分离的结构，在兼顾高效数据包处理的同时，增强其灵活可扩展可编程能力。

问题：路由器如何实现支持虚拟化？

  • 通用计算机系统的虚拟化技术: 执行计算和调度任务

  • 路由器的虚拟化技术: 执行数据包查找与转发任务

在数据平面与控制平面相分离的路由器结构中：

  • 控制平面的虚拟化实现 - 采用当前通用计算机系统的虚拟化技术

  • 数据平面的虚拟化实现 - 起步阶段

相关技术研究进展

虚拟化技术

通用计算机系统的虚拟化技术按虚拟层次分为三类：

  • 全虚拟化

  • 半虚拟化

  • 操作系统级虚拟化

全虚拟化 - 本地虚拟化技术

eg. VMware, KVM etc.

Structure:

Description:

当客户操作系统执行某项特权指令时，中间层将接管该段代码，并执行对应的操作。

Drawbacks:

输入输出(I/O)性能受到一定影响。

半虚拟化 - 准虚拟化技术

eg. Xen, UML

Structure:

Description:

该技术通过修改操作系统代码，让客户操作系统本身知道其运行在一个虚拟化平台上。显著提升虚拟化技术的I/O性能。

Drawbacks:

实现非常复杂。

操作系统级虚拟化

eg. Docker, OpenVZ, Linux-VServer, LXC

Structure:

Description:

利用操作系统所提供的一些机制(如Linux中命名空间)，将一组进程放入到“容器”中，与主操作系统的进程隔离开来，并采用这些机制限制该组进程的CPU和内存占用率，从而实现虚拟化技术中资源隔离与分配。虚拟化开销非常小，非常适合虚拟化路由器这样功能单一且性能需求较高专用设备。

Drawbacks:

只能运行在特定的操作系统上，所有虚拟机共享一个内核，不能运行多个异构的操作系统。

可编程设计

1.XORP:

XORP是旨在提供一个开放的、可扩展的路由器软件平台。

2.Quagga:

路由器软件平台，具有路由收敛更快、兼容性更好、更加轻量化等优点。

3.Click:

针对路由器数据平面的模块化可编程平台，在数据平面上实现灵活的功能配置和模块的可重用性。

数据包转发加速方法

  • 轮询方式 - 内核版本的Click路由器

  • 集群技术 - RouteBricks

  • 路由表压缩算法

  • 通用GPU - PacketShader

可编程虚拟化路由器实例

1.vRouter - 一种基于通用计算机平台的纯软件虚拟路由器

  • 硬件多队列特性 <=> 不同数据平面的数据隔离

  • Xen虚拟技术 <=> 不同的控制平面隔离性

  • 内核态Click的轮询技术 <=> 提高数据包转发吞吐量

Drawbacks:

由于采用纯软件的实现方式，其性能及可扩展性较差。

2.CAFE, SwitchBlade - 可配置的数据转发引擎CAFE, 虚拟路由器的模块化的流水架构SwitchBlade

CAFE采用不同API配置底层硬件，实现不同的数据平面，从而避免了底层硬件的重新设计。

SwitchBlade支持硬件可编程性，从而在确保底层硬件的高速转发性能前提下，支持新路由协议的快速原型实现与部署。

Drawbacks:

只关注硬件数据平面的可编程性，未解决好虚拟路由器结构在性能和可编程行之间折衷问题。

3.OpenFlow

可编程虚拟化路由器平台 - PEARL

1.虚拟化及隔离性

软件层面：操作系统级虚拟机LXC给每个虚拟路由器提供了独立的“容器”，实现共享CPU和内存资源，确保资源分配的可配置性和虚拟机间的相互隔离。

硬件层面：专用硬件数据包处理卡执行上层软件对于各个虚拟路由器数据平面的配置，实现数据通路的资源分配与隔离。

软硬件之间：独立的I/O通道，实现数据的互不干扰。

2.配置和可编程的灵活性

配置 - 提供高优先级和低优先级虚拟路由器实例

  • 高优先级实例：独自享有一个高性能I/O通道，在底层数据包处理卡上拥有独立三态内容寻址存储器(TCAM)空间 - 适合需要处理大流量和复杂操作等网络应用

  • 低优先级实例：共享一个I/O通道，没有专门TCAM硬件用于查找加速 - 适用于小流量的网络应用

可编程 - 插件方式 或 Socket API

  • 插件方式：直接嵌入其代码在数据包处理的守护进程中，性能高

  • Socket API：具备更好的通用性

3.高效数据包查找与转发

  • 专用数据包处理卡采用高速PCIE总线接口 - 极高的数据吞吐能力

  • 内嵌的TCAM和SRAM芯片 - 高速复杂查找等功能
